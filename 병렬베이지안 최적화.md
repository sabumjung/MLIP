병렬 베이지안 최적화 환경에서 UCB, Greedy Pending, Kriging Believer, qLogEI는 제한된 실험 횟수 내에 최적해를 신속하게 찾기 위해 한 번에 여러 개의 실험 지점(Batch)을 선택하는 획득함수(Acquisition Function)이다.
4가지 방법은 병렬 처리 시 발생하는 '중복 선택' 문제를 해결하고 탐색(Exploration)과 활용(Exploitaiton)의 균형을 맞추는 방식에서 각각 고유한 수치적 메커니즘을 갖는다.

- 목표 : 전해질의 저온 전도도를 최대화한다.
- 현상태 : 5번의 실험 진행했고, 그 중 최고 성능(Best Value, $f_{best}$)은 0.85이다.
- 모델 예측 : 아직 실험하지 않은 3개의 후보 지점(A, B, C)에 대해 가우시안 프로세스 모델이 다음과 같은 예측 값을 내놓았다.

|지점|예측평균(μ)|예측표준편차(σ, 불확실성)|
|---|---|---|
|지점 A|0.80|0.05|
|지점 B|0.75|0.15|
|지점 C|0.70|0.20|

- 병렬배치 크기(q) : 한 번에 2개 지점을 선택하여 실험을 한다.
- 각 방법론이 위 수치를 바탕으로 다음 실험 지점을 어떻게 선택하는지 비교해보겠다.

# UCB(Upper Confidence Bound)
UCB는 각 후보 지점의 예측 평균에 불확실성(표준편차)을 가중치(K)하여 더한 '상한 신뢰구간'을 점수로 사용한다. 일반적으로 K는 1~2사이의 값을 사용하며, 여기서는 탐색을 강조하기 위해 K=2로 설정한다.

수식 : Score(x) = $μ(x) + 2 × σ (x)$
각 지점의 점수 계산
- 지점 A : 0.80 + 2×0.05 = 0.90
- 지점 B : 0.75 + 2×0.15 = 1.05
- 지점 C : 0.70 + 2×0.20 = 1.10

> 선택결과 : 점수가 높은 순서대로 지점 C(1.10)와 지점 B(1.05)를 선택한다. 이 방식은 평균 성능이 가장 높은 A를 배제하고, 불확실성이 큰 C와 B를 선택함으로써 새로운 영역(탐색)을 적극적으로 조사한다. 병렬환경에서는 이렇게 선택된 두 지점이 서로 너무 가깝지 않도록 추가적인 다중 무장 강도(MAB) 정택을 적용하기도 한다.

# Greedy Pending
그리디 대기 방식은 아직 결과가 나오지 않은 실험들을 고려하여, 현 시점에서 즉시 선택할 수 있는 가장 유망한 지점을 탐욕적으로 선택하는 방법이다.

> 작동원리 : 이 방법은 현재 진행 중인 실험이 없다고 가정하고, 획득함수를 최대화하는 지점을 순차적으로 q개 만큼 선택한다.

- 1번째 선택 : 위 UCB 점수 계산 결과, 가장 높은 지점 C(1.10)를 첫번째로 선택한다.
- 2번째 선택(Pending 고려)
	- 만약 단순히 다시 점수를 계산하면 지점 C와 B가 다시 선택될 수 있어 중복 실험이 발생한다.
	- Greedy Pending은 첫 번째로 선택된 C를 '대기 중(Pending)'으로 표시하고, C를 제외한 나머지 지점 중에서 점수가 가장 높은 지점 B(1.05)를 두 번째로 선택한다.
	- 혹은, C를 선택했을 때의 가상 관측값(예 : 평균값 0.70)을 모델에 임시로 반영하여 업데이트한 후, 업데이트 된 모델에서 다음 지점을 찾기도 한다.
 
> 선택결과 : 지점 C와 B가 선택된다. 이 방법은 계산 복잡도가 낮아 비동기 병렬 처리에 적합하지만, 다양성을 명시적으로 장려하지 않을 경우 과탐색이 발생할 수 있어 DPP(Determinantal Point Process) 등의 기법과 결합되기도 한다.

# Kriging Believer
Kriging Believer는 병렬로 선택된 지점들에 대해 가상의 관측값(Fantasy Data)을 부여하여 모델을 업데이트한 후, 업데이트 된 모델을 기반으로 다음 지점을 선택하는 휴리스틱이다.

> 작동원리 :

- 가상의 관측값 할당 : 선택 후보인 지점 C와 B에 대해, 실제 실험 결과 대신 모델이 예측한 평균값$(μ)$을 관측된 것처럼 가정한다.
	- 지점 C에 가상 관측값 $y_{fake} = 0.70 부여
	- 지점 B에 가상 관측값 $y_{fake} = 0.75 부여

- 모델 업데이트 : 이 가상 데이터를 기존 데이터셋에 추가하여 GP 모델을 재학습(업데이트)한다.
	- 업데이트 과정에서 지점 C와 B주변의 불확실성 $(σ)$이 급격히 감소한다.
 
- 다음지점 선택 : 업데이트된 모델에서 획득 함수(예 : UCB)를 다시 계산하여 q개의 지점을 선택한다.

> 수치적 효과

- 가상 관측값이 부여되면, 지점 C와 B의 $σ$값이 0에 가까워지므로, 해당 지점들의 UCB 점수는 $μ$값 수준으로 떨어진다.
- 반면, 아직 가상 데이터가 부여되지 않은 지점 D(가정 : μ = 0.65, σ = 0.30)가 있다면, 업데이트된 모델에서 지점 D의 불확실성이 상대적으로 높게 유지된다.
- 결과적으로 KB는 C와 B를 선택한 후, 이들이 '이미 실험된 것'으로 간주되도록 만들어, 다음 배치에서는 완전히 새로운 지점(예 : D)을 선택하도록 유도한다.

> 이 방식은 명시적인 다양성 페널티를 추가하지 않아도, 가상 관측을 통한 모델 업데이트 자체가 중복 선택을 방지하는 다양성 메커니즘으로 작동한다.





# qLogEI
qLogEI는 병령 배치 최적화를 위해 설계된 획득함수로, 기존 기대 개선량(EI)의 수치적 불안정성을 해결하고 병렬 선택의 효율성을 극대화한 방법이다.
- 수치적 작동 원리 : qEI는 q개의 지점이 동시에 선택되었을 때의 기대 개선량을 계산하지만, 수치적 불안정성이 발생할 수 있다. qLogEI는 개선량에 로그(log)를 취하고 소프트플러스(softplus) 함수를 적용하여 수학적 양정성을 보장하고 그래디언트가 사라지는 문제를 완화한다.

- 병렬 최적화 방식 : qLogEI는 단일 지점씩 선택하는 그리디 방식이 아닌, q개의 지점을 공동으로 최적화한다. 이는 q점 결합 확률 밀도 함수에 대해 획득 함수를 적분하여 최대화하는 방식으로 몬테카를로 샘플링을 통해 근사한다.

- 성능비교 : 노이즈가 있는 문제(예 : Hartmann 함수)에서 qLogEI는 UCB/LP나 qUCB보다 빠른 수렴 속도를 보이며, 특히 고차원 문제나 노이즈가 있는 실제 데이터 기반 모델에서 우수한 성능을 발휘한다.

> 예


|지점|예측평균(μ)|예측표준편차(σ, 불확실성)|
|---|---|---|
|지점 D|0.84|0.01|
|지점 E|0.70|0.15|
|지점 F|0.65|0.20|

- 기존 qEI의 수치적 문제 : 표준 qEI는 E[max(max(Y)-$f_{best}$, 0)]를 계산한다. 여기서 Y는 모델 예측 분포이다.
- 지점 D의 경우, $mu=0.84$로, $f_{best}$ = 0.85보다 낮다 개선량은 max(0.84 - 0.85, 0)=0이 된다.
- 지점 E, F도 모두 qEI가 0에 수렴한다. 따라서 최적화기는 그래디언트 기반 탐색이 불가해져 무작위 샘플링이나 초기 조건에 의존하게 된다.
- 컴퓨터 연산에서 평균값과 현 최적화값이 매우 가까울 때, 개선량 계산 과정에서 그래디언트가 0으로 소실되는 문제가 발생한다. 이는 최적화 알고리즘이 "더 이상 개선될 여지가 없다"고 판단하여 탐색을 멈추게 만들거나, 수치적 오차로 인해 잘못된 지점을 선택하게 한다.
- 특히 고차원 문제나 노이즈가 있는 환경에서는 이러한 수치적 병목 현상이 심화되어 qEI가 무작위 탐색과 유사한 성능을 보이기도 한다.

> 해결방법 : qLogEI의 수치적 해결 메커니즘

qLogEI는 BoTorch라이브러리에서 구현되며, 다음과 같은 방법으로 문제를 해결한다.
- 소프트 플러스 적용 : 개선량 max(max(Y)-$f_{best}$,0) 대신 softplus(max(Y)-$f_{best})를 사용한다. softplus(x) = log(1+$e^x$)로 정의되며, x가 음수일 때도 0이 아닌 작은 양수 값을 유지한다. 이는 그래디언트가 완전히 0이 되는 것을 방지한다.
- 로그 변환 : 개선된 값에 로그를 취하여 log(softplus( ... )) 형태를 만든다. 이는 확률적 개선량의 분포를 안정화하고, 최적점 근처에서의 그래디언트 크기를 적절히 조절한다.
- 공동 최적화 : q개의 지점 X=(x1, ..., xq_를 별도의 변수로 취급하여 몬테카를로 샘플링을 통해 결합 사후 분포에서 직접 최대화한다. 이는 순차적 그리디 방식보다 전역 최적점을 찾을 확률을 높인다.)
